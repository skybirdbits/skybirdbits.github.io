توضیحات

سطح دسترسی پیشفرض تمام اعضا در کاتلین public است.

با استفاده از کلیدواژه های سطح دسترسی (access modifiers) در کاتلین میتونیم مشخص کنیم متغیرها، توابع، کلاس ها و اینترفیس ها کجاها قابل دسترسی اند.

نکته

کلیدواژه های سطح دسترسی برای متغیر ها، توابع و کلاس های محلی (local) قابل استفاده نیستند.

در کاتلین چهارتا کلیدواژه ی سطح دسترسی داریم:

۱- public

۲- protected

۳- internal

۴- private

سطح دسترسی برای اعضای تاپ لول

متغیر ها، توابع، کلاس ها، آبجکت های اعلامی و اینترفیس ها میتونن به صورت top level تعریف بشن.

private

با کلیدواژه ی private اعلام میکنیم عضو فقط برای اعضای داخل فایل در دسترس است.

private var v = ... private fun myFunction(){ ... } private class MyClass{ ... }

internal

با کلیدواژه ی internal میگیم عضو فقط برای اعضای داخل ماژول (module) در دسترس است.

internal var v = ... internal fun myFunction(){ ... } internal class MyClass{ ... }

public

با استفاده از کلیدواژه ی public اعلام میکنیم کل پروژه میتونه به عضو دسترسی داشته باشه.

public var v = ... public fun myFunction(){ ... } public class MyClass{ ... }

نکته

کلیدواژه ی protected برای اعضای top-level قابل استفاده نیست.

سطح دسترسی برای اعضای کلاس

private

با استفاده از کلیدواژه ی private برای یک عضو کلاس اعلام میکنیم فقط اعضای داخل کلاس میتونن بهش دسترسی داشته باشن.

class MyClass{ private var prop = ... ... private fun myFunction(){ ... } ... }

protected

اگه از کلاسی که عضوش با سطح دسترسی protected تعریف شده ارث بری نداشته باشیم مثل private عمل میکنه. در صورت ارث بری کلاس فرزند به عضوی که در کلاس پدر با protected تعریف شده دسترسی داره.

class MyClass{ protected var prop = ... ... protected fun myFunction(){ ... } ... }

internal

تمام اعضای module که به کلاس دسترسی دارن؛ به عضو کلاس که با سطح دسترسی internal تعریف شده نیز دسترسی دارن.

class MyClass{ internal var prop = ... ... internal fun myFunction(){ ... } ... }

public

کل پروژه به عضوی که در کلاس با سطح دسترسی public تعریف شده دسترسی داره.

class MyClass{ public var prop = ... ... public fun myFunction(){ ... } ... }

خلاصه

- کلیدواژه های سطح دسترسی تعیبن میکنن یک تابع متغیر، کلاس و ... در چه محدوده ای قابل دسترسی هستند.

- در کاتلین چهارتا کلیدواژه ی سطح دسترسی داریم که عبارتند از private، protected، internalو public.

- کلیدواژه های سطح دسترسی برای اعضای داخلی (local) قابل استفاده نیستند.

- کلیدواژه ی protected برای اعضای top level قابل استفاده نیست.
